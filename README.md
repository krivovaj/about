# Julia Krivova — personal website

Одностраничный сайт-визитка Юлии Кривовой о карьере в Talent Acquisition, Talent Programs и Program/Project Management.

Сайт работает без фреймворков и сборки: достаточно открыть `index.html` или запустить локальный веб-сервер.

## Возможности

- английская и русская версии;
- яркий и лаконичный стили;
- адаптация для компьютеров и мобильных устройств;
- карьерный таймлайн, проекты и отзывы;
- интерактивная кнопка **Surprise me / Удиви меня**;
- ссылки на email, LinkedIn и проекты.

## Структура проекта

```text
site-vizitka/
├── assets/
│   └── julia-krivova.jpeg  # фотография
├── index.html              # содержание страницы
├── styles.css              # оформление и адаптивность
├── script.js               # переключатели и случайные факты
└── README.md
```

Папка `tmp/` содержит рабочие материалы и для публикации не нужна.

## Как открыть локально

Самый простой способ — дважды нажать на `index.html`.

Надёжнее запустить локальный сервер. В терминале перейдите в папку проекта и выполните:

```bash
python3 -m http.server 4173
```

Затем откройте [http://127.0.0.1:4173](http://127.0.0.1:4173). Остановить сервер можно сочетанием `Ctrl+C`.

## Как редактировать

- Тексты, ссылки, опыт, проекты и отзывы находятся в `index.html`.
- Русские переводы хранятся в атрибутах `data-ru` рядом с английским текстом.
- Стили находятся в `styles.css`.
- Случайные факты находятся в массивах `facts.en` и `facts.ru` файла `script.js`.
- Чтобы заменить фотографию, положите новое изображение в `assets/` и измените путь в `index.html`.

После правок проверьте обе языковые версии, оба стиля и мобильный вид страницы.

## Бесплатная публикация: Cloudflare Pages

Это самый быстрый вариант, если не хочется работать с Git.

1. Создайте бесплатный аккаунт на [Cloudflare](https://dash.cloudflare.com/).
2. Откройте **Workers & Pages**.
3. Выберите **Create application → Get started → Drag and drop your files**.
4. Укажите название проекта, например `julia-krivova`.
5. Загрузите `index.html`, `styles.css`, `script.js` и папку `assets` вместе с её содержимым. Можно также загрузить отдельную папку, содержащую только эти файлы.
6. Нажмите **Deploy site**.

Сайт получит бесплатный адрес вида:

```text
https://julia-krivova.pages.dev
```

Для следующего обновления откройте проект в Cloudflare Pages, выберите **Create a new deployment** и загрузите обновлённые файлы. Не загружайте папку `tmp/`.

Официальная инструкция: [Cloudflare Pages — Direct Upload](https://developers.cloudflare.com/pages/get-started/direct-upload/).

## Альтернатива: GitHub Pages

Этот вариант удобен, если сайт будет регулярно обновляться.

1. Создайте аккаунт на [GitHub](https://github.com/).
2. Создайте новый публичный репозиторий, например `site-vizitka`.
3. Загрузите в корень репозитория `index.html`, `styles.css`, `script.js`, `README.md` и папку `assets`. Папку `tmp/` загружать не нужно.
4. Откройте **Settings → Pages**.
5. В разделе **Build and deployment** выберите **Deploy from a branch**.
6. Выберите ветку `main`, папку `/ (root)` и нажмите **Save**.

Через несколько минут сайт появится по адресу:

```text
https://USERNAME.github.io/site-vizitka/
```

Если назвать репозиторий `USERNAME.github.io`, адрес будет короче:

```text
https://USERNAME.github.io/
```

GitHub Pages бесплатно работает с публичными репозиториями на тарифе GitHub Free. Обратите внимание: опубликованный сайт доступен всем, поэтому в проекте не должно быть закрытых документов, паролей или других конфиденциальных данных.

Официальная инструкция: [GitHub Pages Quickstart](https://docs.github.com/en/pages/quickstart).

## Перед публикацией

- проверьте email и все внешние ссылки;
- убедитесь, что фотографию можно публиковать открыто;
- не публикуйте исходный CV и содержимое папки `tmp/`;
- откройте сайт на телефоне и компьютере;
- проверьте переключение языка и стиля;
- нажмите кнопку **Surprise me / Удиви меня** несколько раз.
